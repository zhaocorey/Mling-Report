## 二、高度相关专利（直接竞争）

### 2.1 CN117436505B — 百度 — 训练数据处理方法

| 字段 | 内容 |
|------|------|
| **公开号** | CN117436505B |
| **标题** | 训练数据处理方法、训练方法、装置、设备、介质 |
| **申请人** | 北京百度网讯科技有限公司 |
| **优先日** | 2023-10-31 |
| **授权日** | 2024-11-26 |
| **状态** | 有效（已授权） |

**技术方案**：
- 对指令数据进行语义聚类，形成多个指令数据簇；
- 对每个簇内的指令数据进行子聚类，获得多个子簇；
- 从子簇中选择目标指令数据用于模型训练。

**与 M-DaQ 的差异**：
| 差异点 | 百度方案 | M-DaQ |
|--------|----------|-------|
| 质量评估 | 无独立质量评分模块 | QSM（基于 triplet loss 的质量评分模型） |
| 选择策略 | 纯聚类采样（多样性优先） | 两阶段：质量优先 → 多样性补充 |
| 多语言支持 | 未明确 | 18 种语言，语言无关 |
| 复杂度 | 未披露 | O(n log n) |

**威胁等级**：⭐⭐⭐ 中高（聚类选择思路部分重叠，但缺少质量维度）

| 字段 | 内容 |
|------|------|
| **论文** | Zhou et al., "LIMA: Less Is More for Alignment", NeurIPS 2023 |
| **核心思想** | 1K 高质量样本即可达到与 52K 样本相当的微调效果 |
| **数据选择方法** | 按响应长度排序，选择最长的 top-k |
| **专利状态** | **未发现相关专利** |

**分析**：LIMA 提出了"少量高质量"的理念，但未提供自动化质量评分方法，仅按长度启发式选择。M-DaQ 的 QSM 是对 LIMA 理念的技术实现升级。

## 六、新颖性/创造性评估总结

### 6.1 与最近似现有技术的对比

| 技术特征 | M-DaQ | 最近似现有技术 | 差异是否充分 |
|----------|-------|--------------|-------------|
| 多语言 IFT 数据选择 | ✅ 18 种语言 | CN119514633A（中英双语，无选择策略） | ✅ 充分 |
| QSM（triplet loss 质量评分） | ✅ | CN120873113A（IFD 评分，非 triplet） | ✅ 充分 |
| DAS（两阶段质量+多样性选择） | ✅ | CN117436505B（聚类选择，无质量）+ CN120354939A（加权融合，O(n²)） | ✅ 充分 |
| 专家修订正负样本对构建 | ✅ | 无 | ✅ 全新 |
| MMR → 离散化映射 | ✅ | 无 | ✅ 全新 |
| SAH 多语言验证 | ✅ | 无 | ✅ 全新（但可能不构成技术特征） |

### 6.2 新颖性结论

**M-DaQ 在以下方面具有明确的新颖性和创造性**：

1. **QSM + DAS 的整体流水线** — 无任何现有技术同时公开质量评分模型和多样性感知选择的联合框架
2. **基于 triplet loss 的多语言质量评分** — 与所有现有质量评分方法（IFD、LLM-as-judge、长度启发式）均不同
3. **MMR 离散化实现** — 将连续 λ 参数转化为 n_quality / n_diversity 比例，实现 O(n²) → O(n log n) 的复杂度降低
4. **多语言特化** — 18 种语言覆盖，语言无关设计

### 6.3 潜在风险

| 风险 | 等级 | 说明 |
|------|------|------|
| 百度 CN117436505B 的聚类选择 | ⭐⭐⭐ | 权利要求可能覆盖"基于聚类的 IFT 数据选择"的广义概念 |
| 复旦 CN120354939A 的质量+多样性联合 | ⭐⭐⭐ | 权利要求可能覆盖"联合优化质量与多样性"的广义概念 |
| LIMA/AlpaGasus 论文作为现有技术 | ⭐⭐ | 虽无专利，但论文公开构成现有技术，限制权利要求范围 |
| 华为自身论文公开 | ⭐⭐⭐⭐ | M-DaQ 论文在 SIGIR '26 公开后成为自身专利的现有技术，需在公开前申请 |

## 七、建议

### 7.1 权利要求撰写建议

1. **独立权利要求应强调多语言维度** — 这是与所有现有技术的最大差异点
2. **QSM 的 triplet loss 训练方法应作为独立从属权利要求** — 无任何现有技术公开
3. **DAS 的两阶段离散化策略应重点保护** — 与复旦的连续加权方案形成明确区分
4. **避免过度宽泛的权利要求** — 百度和复旦已有聚类选择和质量+多样性的部分布局
