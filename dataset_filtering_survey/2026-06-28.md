### 1. [propella-1: Multi-Property Document Annotation for LLM Data Curation at Scale](https://arxiv.org/abs/2602.12414)

- **作者**: Idahl, Maximilian; Droste, Benedikt; Plüster, Björn; Harries, Jan Philipp | **年份**: 2026
- **方法摘要**: 本文挑战了当前数据策划中「用单一标量分数衡量文档质量」的主流做法。作者训练了一系列小型多语言 LLM（0.6B–4B 参数），能够对文档进行 **18 个属性维度、6 个类别** 的细粒度标注。最引人注目的是其 4B 模型在标注一致性上**超过了更大的通用模型**，且原生支持 **57 种语言**——这使其成为目前覆盖语言最广的多维质量标注系统。
- **关键方法**: `multi-property annotation` `18 quality dimensions` `multilingual annotation (57 languages)` `small LLM annotators`
- **实用价值**: ⭐⭐⭐⭐⭐ — **对多语言数据集构建的核心参考价值**。理由：(1) 57 语言原生支持直接匹配我们的多语言需求；(2) 18 维度标注远超 Meta-Rater 和 QuRating 的 4-5 维度，能捕获更细粒度的质量信号；(3) 小型 LLM（0.6B-4B）的推理成本远低于用大模型做 QuRating 式标注。**建议立即精读 Section 3-4（属性定义和模型训练方法），评估是否可替代 Meta-Rater 作为我们的质量评估层。**

### 2. [Autodata: An Agentic Data Scientist to Create High Quality Synthetic Data](https://arxiv.org/abs/2606.25996)

- **作者**: Kulikov, Ilia; Whitehouse, Chenxi; Wu, Tianhao; Nie, Yixin; Saha, Swarnadeep; Helenowski, Eryk | **年份**: 2026
- **方法摘要**: 提出了一种让 AI Agent 充当「数据科学家」的通用方法——Agent 不仅生成数据，还负责**元优化自身的数据构建策略**。核心创新在于 Agentic Self-Instruct：Agent 通过迭代式反馈循环自我改进数据生成 prompt 和过滤规则。在 CS 研究、法律推理和数学推理三个任务上验证了方法的有效性，展示了 Agent 生成的数据在质量上可与人工标注竞争。
- **关键方法**: `agentic data construction` `meta-optimization of data agents` `Agentic Self-Instruct` `synthetic data generation`
- **实用价值**: ⭐⭐⭐⭐⭐ — 对多语言 SFT 数据的合成生成有直接参考价值。Agentic Self-Instruct 的元优化思路可用于设计多语言合成数据 pipeline——让 Agent 自动为每种语言优化生成策略，而非为每种语言手动设计 prompt。**建议精读 Section 4（Agentic Self-Instruct 方法），评估其在多语言 SFT 数据构建中的可行性。**

### 4. [DataShield: Safety-degrading Data Filtering for LLM Benign Instruction Fine-Tuning](https://arxiv.org/abs/2606.00160)

- **作者**: Zhang, Junbo; Zhou, Qianli; Deng, Xinyang; Jiang, Wen; Pan, Jie; Zhu, Jinbiao | **年份**: 2026
- **方法摘要**: 揭示了一个常被忽视的问题：**即使是被标记为"良性"的微调数据集，也可能包含降低 LLM 安全能力的样本**。作者提出了"安全退化评分"（safety degradation score），量化每个样本对模型合规行为的贡献程度。这是一种以安全为中心的数据过滤方法，与基线中已有的安全数据分类（如 Multilingual Safety Classification）形成互补——后者关注"过滤有害数据"，DataShield 关注"过滤可能削弱安全能力的良性数据"。
- **关键方法**: `safety degradation scoring` `benign data filtering` `compliance behavior analysis` `data-centric safety`
- **实用价值**: ⭐⭐⭐⭐⭐ — **对多语言安全数据构建有战略价值**。理由：(1) 在多语言 SFT 中，某些语言的"良性"数据可能无意中削弱模型的安全对齐；(2) Safety degradation score 提供了一种可量化的方法来识别这类"隐性有害"样本；(3) 可与 Data-Quality Illusion 的发现结合——分类器过滤不仅可能无效，还可能有害（安全维度）。**建议与基线中 Data-Quality Illusion 论文联合精读。**

### 5. [Rethinking Data Curation in LLM Training: Online Reweighting Offers Better Generalization than Offline Methods](https://arxiv.org/abs/2605.05227)

- **作者**: Zhao, Wanru; Chen, Yihong; Tang, Yuzhi; Ma, Wentao; Hu, Shengchao; Hu, Shell Xu | **年份**: 2026
- **方法摘要**: 系统性地论证了**离线数据选择/混合方法会牺牲数据多样性、损害模型泛化能力**这一核心问题。作者提出将数据策划重新定义为「在线重加权」——在训练过程中动态调整每个样本的重要性权重，而非在训练前一次性选择或混合。实验证明在线重加权在多个泛化指标上优于离线方法。
- **关键方法**: `online reweighting` `dynamic sample importance` `generalization-focused curation` `training-integrated curation`
- **实用价值**: ⭐⭐⭐⭐⭐ — **对数据混合策略有范式性影响**。这是本周期内 4 篇「在线/动态选择」论文中最具理论深度的一篇。其核心洞察——离线方法牺牲多样性——对多语言数据混合尤其重要：如果我们预先确定语言比例（离线），可能会丧失模型在训练过程中自主发现某些语言价值的能力。**建议精读 Section 3（离线 vs 在线的理论分析），评估其对我们 Topic-based mixing 策略的影响。**

### 6. [UniGeM: Unifying Data Mixing and Selection via Geometric Exploration and Mining](https://arxiv.org/abs/2602.03772)

- **作者**: Wang, Changhao; Yu, Yunfei; Yao, Xinhao; Yang, Jiaolong; Cantaro, Riccardo; Li, Chaobo | **年份**: 2026
- **方法摘要**: 提出了一种**将数据混合和数据选择统一为流形近似问题**的分层框架。宏观层（Macro-Exploration）通过稳定性聚类学习混合权重；微观层（Micro-Mining）基于几何距离过滤高质量样本。最引人注目的实验结果：在 100B token 规模上实现了 **2× 的数据效率**——即用 50B token 就能达到基线 100B 的训练效果。
- **关键方法**: `unified mixing+selection` `manifold approximation` `hierarchical clustering` `geometric distance filtering`
- **实用价值**: ⭐⭐⭐⭐⭐ — 2× 数据效率的提升极为惊人，且「混合 + 选择统一化」的思路与基线中 HDS（RL 混合）和 MATES（模型感知选择）各自独立优化的路线形成了第三条路径。**如果其几何方法能迁移到多语言场景，可能同时解决我们的混合和选择问题。建议精读 Section 3-4（Macro-Exploration 和 Micro-Mining 的数学形式化）。**

### 7. [HARP: Efficient Data Selection for Finetuning Large Language Models](https://arxiv.org/abs/2606.07690)

### 12. [OPUS: Towards Efficient and Principled Data Selection in Large Language Model Pre-training in Every Iteration](https://arxiv.org/abs/2602.05400)

### 15. [IGDS: From Insight to Action: Interpretability-Guided Data Selection in Large Language Models](https://arxiv.org/abs/2604.25167)

- **作者**: Shi, Ling; Wu, Xinwei; Zhao, Xiaohu; Wang, Hao; Liu, Heng; Liu, Yangyang | **年份**: 2026

## 📊 趋势洞察

### 一、在线/动态选择趋势的加速确认

基线报告（v2, 2026-06-24）已将 Model-Aware Selection 识别为主流范式。本次增量发现**进一步加速了这一趋势**，且从「模型感知」深化为「优化器感知」和「在线重加权」：

| 基线方法（2024-2025） | 新增方法（2026） | 演进方向 |
|----------------------|-----------------|---------|
| MATES（模型感知选择） | OPUS（优化器诱导空间选择） | 从模型状态 → 优化器状态 |
| HDS（RL 在线混合） | Filter-then-Weight（优化器感知在线选择） | 从混合 → 选择+重加权 |
| Quad（影响力+多样性） | GRACE（图引导动态 coreset） | 从静态 → 动态 coreset |
| — | Rethinking Data Curation（在线重加权理论） | 新增理论基础 |

**对多语言数据集构建的启示**: 多语言数据选择不能是一次性的。模型在训练过程中对不同语言的学习进度不同——某些语言可能在早期就被充分学习，而另一些语言需要更长时间的训练。**在线动态选择允许模型在训练过程中自动调整各语言的数据配比**，这比预先确定的静态比例更有效。

### 二、智能体化数据构建从概念走向实践

基线中有 Multi-Agent Collaborative Selection 和 Multi-Actor Collaboration 两篇关于多智能体数据选择的论文，但它们聚焦于**选择**已有数据。本次增量中，Autodata 和 OpenThoughts-Agent 将智能体应用推向了**数据构建（construction）**——让 AI Agent 不仅选择数据，还生成和优化数据。

```
基线: 智能体 → 选择数据 (Multi-Agent Selection)
新增: 智能体 → 构建数据 (Autodata, OpenThoughts-Agent)
未来: 智能体 → 端到端数据 pipeline (推测)
```

**对多语言数据集构建的启示**: 多语言数据的稀缺性问题可能部分通过 Agentic 合成来解决。Autodata 的 Agentic Self-Instruct 方法允许 Agent 为每种语言自动优化合成策略，这比手动为每种语言设计 prompt 更高效。OpenThoughts-Agent 的 100+ 消融实验提供了评估 agentic 数据 pipeline 各阶段效果的方法论。

### 三、安全数据过滤的兴起

DataShield 是一个重要的新信号。基线中已有安全相关的数据工作（如 Multilingual Safety Classification、VityaVitalich 安全数据），但它们聚焦于**过滤有害内容**。DataShield 揭示了一个更微妙的问题：**即使内容本身无害，某些数据也可能削弱模型的安全对齐**。

这对多语言场景尤其重要：
- 不同语言对"安全"的文化定义不同
- 某些语言的安全训练数据可能不足，导致模型在这些语言上的安全对齐较弱
- DataShield 的 safety degradation score 提供了一种量化方法来识别这类风险

### 四、影响函数效率的持续突破

基线报告已指出 EMNLP 2025 对 IF 在 LLM 上适用性的质疑，并建议采用 Proxy Models 替代路径。本次增量中 3 篇论文（IPROX、PRISM、HARP）从不同角度回应了这一挑战：

| 方法 | 策略 | 与基线的关系 |
|------|------|------------|
| IPROX | 影响力保持代理（小模型替代大模型计算 IF） | 直接回应 EMNLP 2025 质疑 |
| PRISM | 偏好感知 IF（加权目标样本） | 扩展 IF 的应用场景 |
| HARP | 层级剪枝避免反复微调 | 绕过 IF 计算，直接评估效用 |

**对多语言数据集构建的启示**: IF proxy 方法在多语言场景下的可靠性是一个关键开放问题。建议在我们构建多语言 pipeline 时，系统性评估 IPROX 代理在不同语言上的表现是否一致。
