### 1. [Holistic Data Scheduler (HDS): Online Data Mixing via Multi-Objective Reinforcement Learning](https://arxiv.org/abs/2606.24133)

- **作者**: 详见 arXiv | **年份**: 2026-06 | **类别**: online_data_mixing
- **方法摘要**: 将在线数据混合建模为连续控制空间的强化学习问题，使用 Soft Actor-Critic (SAC) 策略。引入多目标整体奖励，融合数据驱动的质量信号、损失驱动的跨域影响和模型驱动的权重范数奖励。在 The Pile 上以比次优方法 **少 44% 的训练迭代** 达到相同验证困惑度，MMLU 提升 7.2%。
- **关键方法**: `RL-based online data mixing` `Soft Actor-Critic` `multi-objective holistic reward` `continuous control space`
- **实用价值**: ⭐⭐⭐⭐⭐ — **对多语言数据混合策略有核心参考价值**。将数据混合从静态比例分配提升为在线动态决策，44% 的训练效率提升极为显著。多目标奖励设计（质量 + 跨域影响 + 模型状态）可直接迁移到多语言混合场景——不同语言域之间的相互影响正是多语言混合的核心难题。**建议精读 Section 3-4（SAC 建模和多目标奖励设计），评估其多语言适配性。**

### 2. [Learning at the Right Pace: Adaptive Data Scheduling Improves LLM Reinforcement Learning (ADS)](https://arxiv.org/abs/2606.22305)

- **作者**: 详见 arXiv | **年份**: 2026-06 | **类别**: adaptive_data_scheduling
- **方法摘要**: 提出双层自适应数据调度框架替代 RL 后训练中的均匀采样。集群层维护语义集群间的自适应分布，样本层选择策略边界样本以产生信息性优势。在 3 个 LLM 和 7 个推理基准上，相比 GRPO 均匀采样提升 5.2% 平均准确率。
- **关键方法**: `dual-level adaptive data scheduling` `semantic cluster-based pacing` `policy-boundary sample selection` `RL post-training`

### 3. [GRACE: Gradient-aligned Reasoning Data Curation for Efficient Post-training](https://arxiv.org/abs/2605.13130)

- **作者**: 详见 arXiv | **年份**: 2026-05 | **类别**: reasoning_data_curation
- **方法摘要**: 将每条推理轨迹视为一系列优化事件，通过评估每步与答案导向梯度方向的对齐程度和与前序轨迹的一致性来打分。引入表征级梯度代理，仅需单次前向传播即可评估步级对齐度。GRACE 以 **20% 数据达到 108.8% 的全数据性能**，5% 数据保持 100.2% 性能。
- **关键方法**: `gradient-aligned step scoring` `representation-level gradient proxy` `step-level curation` `cross-backbone transfer`
- **实用价值**: ⭐⭐⭐⭐⭐ — **推理数据选择的突破性工作**。步级粒度的策划远超现有样本级方法，20% 数据超越全数据集的效果极为惊人。表征级梯度代理的单次前向传播设计使其在工程上可行。跨骨干迁移能力对多语言推理数据构建尤为重要。**建议精读 Section 3（梯度代理设计），评估其在多语言推理链中的适用性。**

### 7. [Beyond What to Select: Plug-and-play Oscillatory Data-Volume Scheduling (PODS)](https://arxiv.org/abs/2605.14773)

- **作者**: 详见 arXiv | **年份**: 2026-05 | **类别**: data_volume_scheduling
- **方法摘要**: 从优化视角发现数据选择训练引入隐式正则化，且由选择比例调控。提出 PODS：动态调度 **选多少数据**（而非选哪些样本），在低比例正则化和高比例恢复阶段之间振荡交替。ImageNet-1k 训练成本降低 50% 同时提升精度；LLM 指令微调加速 2× 无性能退化。
- **关键方法**: `oscillatory data-volume scheduling` `selection-induced regularization` `ratio-level scheduling` `plug-and-play`
- **实用价值**: ⭐⭐⭐⭐⭐ — **正交于所有现有方法的数据效率优化**。不关心"选什么"而关心"选多少"，可与任何其他选择方法叠加使用。2× 指令微调加速对多语言 pipeline 的工程价值极高——多语言 SFT 通常涉及大量语言和任务，训练成本是主要瓶颈。**即插即用设计，建议作为训练流程的标准组件。**

- **方法摘要**: 研究 LLM 基准测试集的 coreset 选择：选择小的 prompt 子集使其模型评分和排名近似完整基准。使用子模子集选择（DPP、子模互信息、设施位置函数），仅基于语义 prompt 嵌入，无需模型评估结果。在 35 个异构基准、18 个前沿 LLM、61K+ prompts 上验证。
- **关键方法**: `submodular subset selection` `facility location function` `determinantal point processes` `benchmark compression`
- **实用价值**: ⭐⭐⭐ — **基准测试压缩方法**。对多语言评估有参考价值：多语言基准测试通常包含大量语言和任务，coreset 方法可显著降低评估成本。设施位置函数的无评估特性使其不依赖特定模型。

## 📊 趋势洞察

### 一、RLVR 后训练数据选择的集中爆发 📈📈📈

本周期 **6 篇论文** 聚焦 RL/RLVR 后训练阶段的数据选择（ADS、SHIFT、IRDS、LZE、HES 的 RL 部分、课程 RL），配合 **5 个新开源仓库**。这标志着数据选择研究的重心正在从预训练和指令微调阶段向 RL 后训练阶段迁移。

| 方法 | 核心信号 | 训练需求 | 性能提升 |
|------|---------|---------|---------|
| ADS | 语义集群 + 策略边界 | 训练中在线 | GRPO +5.2% |
| SHIFT | 隐状态动态 (RIRS) | **完全免训练** | N/A (选择器) |
| IRDS | SAE 覆盖 + 验证器 | 轻量评估 | +3.9pp |
| LZE | 闭式能量分数 | 训练中在线 | AIME +45.9% |
| HES | Token 熵求和 | **完全免训练** | 跨范式有效 |
| 课程 RL | pass@k 边界 | 训练中在线 | pass@256 +9.8pp |

**对多语言的启示**: RL 后训练在多语言模型中越来越重要（如多语言 RLHF/DPO），但这些方法尚未在多语言场景下验证。将 SHIFT 或 HES 的训练免方法应用于多语言 RLVR 数据选择是一个高价值研究方向。

### 二、训练免 (Training-Free) 选择方法的崛起

SHIFT、HES、Coresets Before Score Sets 三篇独立工作探索了 **完全不需要训练或模型评估** 的数据选择方法：
- **SHIFT**: 推理时隐状态动态 → RLVR 数据选择
- **HES**: Token 级熵分析 → 统一 SFT/RFT/RL 选择
- **Coresets Before Score Sets**: 语义嵌入子模优化 → 基准压缩

这三者使用完全不同的信号（隐状态偏移、token 熵、语义嵌入），但共同指向一个方向：**好的数据选择信号可以从模型的推理过程中直接提取，无需昂贵的训练循环**。

### 三、数据调度从"选什么"到"选多少"的视角拓展

PODS 开创了 **数据量调度** 这一新维度——不关心选哪些样本，而关心每个阶段选多少数据。其发现"选择比例调控隐式正则化"对多语言训练有直接启示：
- 低比例阶段 → 正则化（防止过拟合某些语言）
- 高比例阶段 → 恢复（确保覆盖所有语言）
- 振荡交替 → 两者兼得

这与基线中 HDS 的 RL 混合、ADS 的自适应调度形成互补：**HDS 决定混合比例，ADS 决定训练顺序，PODS 决定数据量**。

### 四、梯度/优化感知的选择方法持续深化

| 论文 | 梯度利用方式 | 创新点 |
|------|-------------|--------|
| GRACE | 表征级梯度代理 | 单次前向传播的步级评估 |
| GIST | SVD 子空间投影 | LoRA 感知的耦合几何 |
| SLAP | Hessian 近似梯度 | 批感知 + 分层采样 |
| D³ | 损失依赖图 | 动态图约束调度 |
| wICI | 上下文影响力 | ICL 视角的数据价值 |
| MTS Analysis | 梯度 SNR 分析 | 元学习失败原因诊断 |

梯度感知方法从早期的 TracIn/IF 发展为多维度、多粒度的精细方法。GRACE 的步级评估和 GIST 的 LoRA 感知代表了两个重要的细化方向。

### 五、可解释/可审计数据选择成为新诉求

IRDS（SAE 基可解释选择）和 Data Attribution（双向梯度归因）反映了社区对 **数据选择透明度** 的需求增长。对多语言数据构建尤为重要：
- 需要解释为什么选择某些语言的数据而忽略其他
- 需要追踪模型行为问题可归因到哪些训练数据源
- 审计能力在多语言合规场景中可能是硬性要求
