### 1. [MuRating: A High Quality Data Selecting Approach to Multilingual Large Language Model Pretraining](https://proceedings.neurips.cc/paper_files/paper/2025/hash/6e54c0fb5c8be96b92055f6a3b6b2d6f-Abstract-Conference.html)

- **会议**: NeurIPS 2025 | **年份**: 2025
- **方法摘要**: 提出了一种可扩展框架，将英语数据的高质量评分信号**跨语言迁移**到多语言预训练数据选择中。核心解决的是低资源语言的质量评估困境——英语训练的质量分类器无法直接应用于其他语言，而为每种语言单独训练分类器又不可行。MuRating 利用英语质量信号作为锚点，通过跨语言对齐机制将质量判断迁移到目标语言。
- **关键方法**: `cross-language quality transfer` `multilingual data selection` `English-anchored quality signals` `scalable multilingual curation`
- **实用价值**: ⭐⭐⭐⭐⭐ — **对多语言数据集构建有核心参考价值，本周期最重要的多语言专项论文。**
  - (1) 直接解决我们的核心痛点：如何为 50+ 语言评估数据质量而不需要为每种语言训练单独的分类器
  - (2) NeurIPS 2025 收录，方法论经过同行评审验证
  - (3) 与 propella-1（57 语言 18 维度标注，06-28 报告）形成互补：propella-1 是多维度标注层，MuRating 是跨语言质量迁移层
  - **建议立即精读，重点评估其跨语言迁移机制在不同语系（汉藏、阿尔泰、南岛等）上的可靠性**

### 3. [DeMix: Decouple Searching from Training — Scaling Data Mixing via Model Merging for LLM Pre-training](https://arxiv.org/abs/2602.00747)

- **会议**: ICML 2026 | **年份**: 2026
- **方法摘要**: 提出用**模型合并**预测最优数据混合比例，无需为每种候选混合方案训练代理模型。核心创新：训练各候选数据集的组件模型，通过加权合并推导混合代理，将搜索成本从 O(混合方案数 × 训练成本) 降至 O(组件模型数 × 训练成本)。同时发布 **DeMix Corpora（22T token 数据集）** 和开源代码。
- **关键方法**: `model merging for mixture prediction` `decoupled search-training` `component model merging` `22T token dataset`
- **实用价值**: ⭐⭐⭐⭐⭐ — **数据混合优化的范式创新，论文+工具+数据集三重价值。**
  - (1) 直接解决多语言混合比例优化问题：无需为每种语言比例方案训练代理模型
  - (2) 22T token 数据集是目前公开的最大规模策划数据集之一
  - (3) ICML 2026 收录，方法论可信度高
  - **建议精读 Section 3（模型合并预测方法），评估其在多语言混合场景中的适用性。DeMix Corpora 应纳入我们的数据集候选清单**

### 4. [The Long-Term Effects of Data Selection in LLM Fine-Tuning (LHAS)](https://arxiv.org/abs/2605.30537)

- **年份**: 2026-05
- **方法摘要**: 首次引入**长周期视角**研究 LLM 数据选择。发现短期最优的选择器存在「排名反转」(rank reversal) 现象——在当前阶段表现最好的选择器，反而会拖慢后续阶段的学习速度并增加灾难性遗忘。作者将其定义为「近视选择」(myopic selection) 问题，并提出 Long-Horizon Aware Selection (LHAS) 目标函数，包含覆盖率、未来代理迁移和反集中度三个正则项。
- **关键方法**: `long-horizon data selection` `rank reversal phenomenon` `myopic selection critique` `LHAS objective` `multi-stage fine-tuning`
- **实用价值**: ⭐⭐⭐⭐⭐ — **对多阶段多语言训练策略有范式性影响。**
  - (1) 多语言训练天然是多阶段的（预训练→SFT→对齐），LHAS 的长期效应分析直接适用
  - (2) "排名反转" 提醒我们：某些语言在早期 SFT 阶段表现好的数据选择方案，可能在后续对齐阶段有害
  - (3) LHAS 的反集中度项可防止模型过度关注少数语言而忽视长尾语言
  - **建议精读 Section 3-4（排名反转变换的数学证明和 LHAS 目标函数设计）**

### 6. [MADS: Model-Aware Diverse Core Set Selection for Instruction Tuning](https://arxiv.org/abs/2605.30857)

- **年份**: 2026-05
- **方法摘要**: 利用 LLM 推理过程中的**神经激活状态**作为模型内在特征进行多样性 coreset 选择。核心发现：3B 模型选择的 core set 可**跨模型迁移**到 7B/8B/13B 模型的微调。在 Alpaca-GPT4（52K 对）上，15% 的 core set 实现全数据集训练 +2.5% 的平均提升。基于覆盖度的选择策略，使用激活特征而非外部嵌入。
- **关键方法**: `neural activation features` `cross-model transferability` `coverage-based selection` `model-intrinsic diversity`
- **实用价值**: ⭐⭐⭐⭐ — **模型内在信号用于数据选择的重要验证。**
  - (1) 跨模型迁移性对多语言场景意义重大：用小模型（如 3B 多语言模型）选择数据，用于大模型（如 70B）训练
  - (2) 神经激活特征不依赖外部嵌入模型，避免了多语言嵌入质量不一致的问题
  - (3) 15% 数据超越全量训练的结论在 Alpaca-GPT4 上验证，但多语言场景需额外验证
  - **建议与 NAIT（#8）联合精读，两者都用模型内在信号但技术路线不同**

### 7. [BRIEF: Bi-level Coreset Selection for Efficient Instruction Tuning in LLMs](https://www.vldb.org/pvldb/vol19/p1264-chai.pdf)

- **会议**: VLDB 2026 | **年份**: 2026
- **方法摘要**: 提出双层优化框架进行 coreset 选择——最小化全数据集与加权 coreset 之间的梯度散度。专门针对 LLM 指令微调中 coreset 选择的过拟合问题。上层优化 coreset 权重，下层优化模型参数，形成 Stackelberg 博弈结构。
- **关键方法**: `bilevel optimization` `gradient divergence minimization` `coreset for instruction tuning` `overfitting mitigation`
- **实用价值**: ⭐⭐⭐⭐ — **指令微调 coreset 选择的理论基础扎实。**
  - (1) 双层优化框架为多语言 SFT 数据选择提供了理论保证
  - (2) VLDB 收录，数据库社区认可其在大规模数据处理上的价值
  - (3) 梯度散度最小化的目标函数在多语言场景中可防止 coreset 偏向某些高梯度语言
  - **建议阅读 Section 3（双层优化公式推导），评估其计算复杂度是否可接受**

### 8. [NAIT: Neuron-Aware Data Selection in Instruction Tuning for Large Language Models](https://arxiv.org/abs/2603.13201)

- **年份**: 2026-03
- **方法摘要**: 通过分析指令微调数据与目标领域能力之间的**神经元激活模式相似性**来评估数据价值。构建可复用、可迁移的神经元激活特征。在 Alpaca-GPT4 上，10% 的子集超越使用外部模型或不确定性特征的方法。关键发现：逻辑推理和编程数据具有最强的跨任务迁移能力。
- **关键方法**: `neuron activation similarity` `reusable activation features` `cross-task transferability` `10% subset beats baselines`
- **实用价值**: ⭐⭐⭐⭐ — **与 MADS 构成「模型内在信号」双子星，但方法论不同。**
  - (1) 神经元激活模式在多语言场景的潜在价值：可识别哪些语言的数据激活了相似的神经元回路
  - (2) 10% 子集超越基线的效率极高
  - (3) "逻辑推理/编程数据跨任务迁移最强"的发现暗示：多语言数据中，推理类数据可能是最通用的训练素材
  - **建议与 MADS 对比阅读，评估激活特征 vs 激活状态在实践中的差异**

### 9. [Active Learning with Low-Rank Structure for Data Selection](https://arxiv.org/abs/2606.16045)

- **会议**: ICML 2026 | **年份**: 2026
- **方法摘要**: 基于低秩近似和残差采样的新 coreset 框架，通过行子集选择问题形式化。选择 O(k + 1/ε²) 个点的加权子集，具有 **(1+ε) 相对误差保证**。利用全局代数结构而非几何聚类。在真实数据集上优于均匀采样和基于聚类的灵敏度采样。
- **关键方法**: `low-rank approximation` `residual-based sampling` `row subset selection` `(1+ε) relative error guarantee`
- **实用价值**: ⭐⭐⭐⭐ — **理论保证最强的 coreset 方法。**
  - (1) (1+ε) 相对误差保证是目前 coreset 方法中最强的理论结果
  - (2) 低秩结构假设在多语言嵌入空间中可能更成立（多语言数据的内在维度可能低于单语言）
  - (3) ICML 2026 收录，理论质量有保障
  - **建议精读其理论分析，评估低秩假设在多语言数据上的成立程度**

### 12. [TBDFiltering: Sample-Efficient Tree-Based Data Filtering](https://arxiv.org/abs/2601.22016)

- **年份**: 2026-01
- **方法摘要**: 基于文本嵌入的层级聚类，自适应选择文档进行 LLM 评估以估计聚类质量。证明了查询效率：在子树纯度假设下，以 O(子树大小) 次查询正确预测质量，无需预先知道子树结构。实验上优于分类器驱动过滤方法。非参数方法。
- **关键方法**: `hierarchical clustering` `adaptive document selection` `query-efficient filtering` `non-parametric`
- **实用价值**: ⭐⭐⭐ — **有理论保证的过滤方法，但实用性待验证。**
  - (1) 非参数特性意味着不需要训练分类器，对低资源语言友好
  - (2) 查询效率的理论保证在大规模多语言语料上可能有显著成本优势
  - (3) 但子树纯度假设在多语言数据上是否成立需要验证
  - **建议快速浏览方法论，作为分类器过滤的替代方案储备**

## 📊 趋势洞察

### 一、Coreset 选择进入「模型内信号」时代

本次增量中最显著的趋势是 coreset 选择方法从**外部信号**（嵌入相似度、影响力函数、多样性度量）转向**模型内在信号**（神经激活状态、神经元激活模式）：

```
2024-2025（基线）: 外部嵌入 + 影响力函数
  → MATES（模型感知，但仍用外部探针）
  → ZCore（零样本，用嵌入距离）
  → DSIR（重要性重采样，用外部特征）

2026（本次增量）: 模型内在激活信号
  → MADS（神经激活状态作为多样性特征）
  → NAIT（神经元激活模式相似性）
  → GRACE（图引导动态更新，已报告）
  → Active Learning Low-Rank（代数结构）
```

**对多语言数据集构建的启示**: 模型内在信号的最大优势是**语言无关性**——神经激活模式不依赖特定语言的嵌入模型或分词器。这意味着：
- 可以用同一个模型为所有语言生成激活特征，避免跨语言嵌入质量不一致的问题
- MADS 的跨模型迁移性（3B→13B）暗示：多语言模型内部的激活模式可能跨语言共享

### 三、数据选择从「近视」到「远视」的范式转移

LHAS 首次形式化了数据选择的长期效应，揭示了一个对所有数据选择方法都有影响的根本性问题：

```
传统范式: 优化当前阶段 → 选择当前最优数据
新范式 (LHAS): 优化多个阶段 → 考虑选择对后续阶段的影响
```

这对多语言训练的影响尤为深远：
- **预训练阶段**: 选择某些语言的数据可能在预训练阶段表现好，但在 SFT 阶段造成语言偏向
- **SFT 阶段**: 短期最优的多语言 SFT 数据可能在 DPO 阶段导致某些语言的对齐质量下降
- **解决方案**: LHAS 的三正则项（覆盖率 + 未来代理迁移 + 反集中度）为多语言多阶段训练提供了理论框架
