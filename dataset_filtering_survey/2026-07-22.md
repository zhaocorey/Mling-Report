## 🆕 新增论文

### 1. PPL-Factory: Task-Aware and Budget-Aware Data Selection from Language Modeling to Reasoning ⭐⭐⭐⭐

- **提交日期**: 2026-07-20（增量窗口内）
- **类别**: Data Selection

**方法摘要**：
提出任务感知（task-aware）和预算感知（budget-aware）的数据选择框架。核心观察是并非所有训练样本对模型学习贡献相等。通过 perplexity-based 信号，在给定计算预算内高效选择高价值训练样本。方法覆盖从语言建模到推理的全场景。

**关键方法**: task-aware selection, budget-aware filtering, perplexity-based scoring

**实用价值评估**: ⭐⭐⭐⭐ — 对多语言数据集构建有直接参考价值。"预算感知"的思路尤其适合我们的场景——在有限的标注/计算资源下，针对不同下游任务选择最优数据子集。与基线中的 BLISS（bilevel data selection）和 LZE（online data selection for RL）形成互补。

### 2. A Systematic Evaluation of Trajectory Data Curation for LoRA Fine-Tuning of Code Agents ⭐⭐⭐⭐

- **作者**: Yunze Han
- **提交日期**: 2026-07-19
- **类别**: Data Curation / Quality

**方法摘要**：
对代码 agent LoRA 微调中的轨迹数据过滤进行了系统性实证研究（基于 Qwen2.5-Code）。核心问题是：在 SFT of open-weight LLMs on expert agent trajectories 这一主流范式下，轨迹数据的质量和数量如何共同影响模型性能？这是一个被严重低估的问题。

**关键方法**: trajectory quality-quantity trade-off analysis, LoRA fine-tuning evaluation, systematic ablation

**实用价值评估**: ⭐⭐⭐⭐ — 虽然聚焦代码 agent，但"数据质量-数量权衡"的方法论可迁移到多语言数据构建。关键 insight：质量过滤并非总是越多越好，存在最优平衡点。与 7/20 报告的 "A Bitter Lesson for Data Filtering" 形成有趣的对比。

### 3. Learning from Synthetic Data without Model Collapse in Iterative Instruction Tuning ⭐⭐⭐⭐

- **作者**: Xiaonan Luo, Yue Huang, Kehan Guo, Ping He, Chuan Zou, Ting Hua, Xiangliang Zhang
- **提交日期**: 2026-07-18
- **类别**: Synthetic Data / Quality

**方法摘要**：
针对迭代指令微调中合成数据导致的模型崩溃（model collapse）问题。现有工作主要研究如何给性能退化设界，但更有意义的目标是在迭代模型进化中主动防止退化。提出了在后续生成 LLM 训练于越来越多模型生成数据时，保证性能提升而非下降的方法。

**关键方法**: anti-collapse training, iterative instruction tuning, synthetic data quality control

**实用价值评估**: ⭐⭐⭐⭐ — 对使用合成数据扩充多语言训练集有直接指导意义。模型崩溃是合成数据路线的核心风险，本文提供了理论和实践上的解决思路。

### 4. DataShield: Uncovering Risky Fine-Tuning Data Across LLMs Through Consensus Subspace Alignment ⭐⭐⭐⭐

- **作者**: Zefeng Wu, Weiwei Qi, Jielong Chen, Tianhang Zheng, Di Hong, Chaochao Lu, Liang He, Zhan Qin, Kui Ren
- **提交日期**: 2026-07-16
- **类别**: Data Quality / Safety

**方法摘要**：
提出跨 LLM 的风险数据评估框架 DataShield。现有方法用单一方向表示安全性（safety direction），限制了风险评估的有效性和可迁移性。DataShield 通过 consensus subspace alignment 识别跨多个 LLM 的风险性微调样本和响应片段，提高了跨模型泛化能力。

**关键方法**: consensus subspace alignment, cross-model risk assessment, safety direction analysis

**实用价值评估**: ⭐⭐⭐⭐ — 对多语言安全数据过滤有参考价值。"共识子空间"方法比单模型评估更鲁棒，可用于识别跨语言/跨模型的有害训练数据。注意：这是与基线中 "DataShield: Safety-degrading Data Filtering for LLM Benign Instruction Fine-Tuning" **不同的论文**（同第一作者但不同合作者和研究侧重）。

### 5. UltraX: Refining Pre-Training Data at Scale with Adaptive Programmatic Editing ⭐⭐⭐⭐⭐

- **作者**: Xinlong Zhao, Dongsheng Liu, Hengyu Zhao, Zixuan Fu, Zheng Wang, Jie Cai, Jie Zhou, Qiang Ma, Xuanhe Zhou, Xu Han, Yudong Wang, Zhiyuan Liu
- **提交日期**: 2026-07-09
- **类别**: Data Filtering / Refinement

**方法摘要**：
提出 UltraX——基于 function-calling 的大规模预训练数据精炼框架。现有方法的局限：规则方法漏检多，LLM 方法效率低且不可靠。UltraX 用自适应程序化编辑（adaptive programmatic editing）替代人工规则或 LLM 打分，实现可编程的数据精炼流水线。来自清华大学（Zhiyuan Liu 团队）。

**关键方法**: function-calling refinement, adaptive programmatic editing, scalable data refinement

**实用价值评估**: ⭐⭐⭐⭐⭐ — **高度实用**。将数据过滤从"写规则"或"跑 LLM"进化到"程序化编辑"，是一个工程友好的范式转换。对多语言数据 pipeline 尤其有价值——可以用统一框架处理不同语言的质量问题，而不必为每种语言写定制规则。

### 6. DRIFT: Refining Instruction Data via On-Policy Data Attribution ⭐⭐⭐⭐

- **作者**: Zefan Wang, Lincheng Li, Tianyu Yu, Yuan Yao
- **提交日期**: 2026-06-16
- **类别**: Data Selection / Attribution

**方法摘要**：
提出 DRIFT（Data Refinement via On-Policy Influence Functions for SFT）。现有 influence function 方法存在两个结构性缺陷：(1) proximity gap——off-policy 验证目标导致的偏差；(2) 严重的梯度范数偏差。DRIFT 不依赖外部参考数据，而是使用 on-policy influence functions 来评估和优化 SFT 训练数据分布。

**关键方法**: on-policy influence functions, data attribution, SFT data refinement

**实用价值评估**: ⭐⭐⭐⭐ — 对 SFT 数据选择有直接应用价值。On-policy 的思路解决了传统 influence function 方法的根本缺陷。与基线中的 "DataFlex" 和 "BLISS" 互补——DataFlex 做动态训练，BLISS 做 bilevel 选择，DRIFT 做 on-policy 归因。

### 7. DataEvolver: Automatic Data Preparation for LLMs through Multi-Level Self-Evolving ⭐⭐⭐

- **作者**: Chao Deng, Shaolei Zhang, Ju Fan, Xiaoyong Du
- **提交日期**: 2026-06-10
- **类别**: Data Curation / Preparation

**方法摘要**：
提出通过多层级自进化（multi-level self-evolving）实现 LLM 训练数据的自动准备。高质量训练数据至关重要但人工标注昂贵。DataEvolver 自动进化数据准备策略，跨多个质量维度进行优化。

**关键方法**: multi-level self-evolving, automatic data preparation, quality optimization

**实用价值评估**: ⭐⭐⭐ — 自动化数据准备的思路有价值，但具体方法论需要进一步验证。对多语言数据构建的自动化有参考意义。

### 8. Learning Multi-Indicator Weights for Data Selection ⭐⭐⭐

- **作者**: Jingze Song, Zihao Chen, Wenqing Chen, Zibin Zheng
- **提交日期**: 2026-05-10
- **类别**: Data Selection

**方法摘要**：
提出联合任务-模型适应框架，学习多个数据质量指标的最优权重。使用高效代理（efficient proxies）避免昂贵的验证。核心贡献是证明学习指标权重比使用固定启发式规则更有效。

**关键方法**: multi-indicator weight learning, efficient proxy evaluation, task-model adaptation

**实用价值评估**: ⭐⭐⭐ — 对多语言数据选择有参考价值，尤其是"学习权重"而非"手动设定"的思路。但在大规模场景下的可扩展性待验证。

### 9. Tailoring the Curriculum: Student-Centered Reasoning Distillation via Dynamic Data-Model Compatibility ⭐⭐⭐

- **作者**: Jiahao Huang, Fei Cheng, Junfeng Jiang, Akiko Aizawa
- **提交日期**: 2026-05-27
- **类别**: Curriculum Learning

**方法摘要**：
提出基于动态数据-模型兼容性的课程学习。不同于固定的难度排序，根据学生模型不断变化的能力自适应调整课程。通过推理蒸馏（reasoning distillation）将大模型的复杂推理能力迁移到小模型。

**关键方法**: dynamic curriculum learning, data-model compatibility, reasoning distillation

**实用价值评估**: ⭐⭐⭐ — 课程学习思路对训练数据调度有参考价值。"数据-模型兼容性"是一个有趣的评估指标，可用于指导多语言训练数据的分阶段投喂。

## 📊 趋势洞察

### 1. 数据选择从"规则驱动"走向"自适应编程"
**UltraX** 代表了一个重要方向转变：不再依赖人工规则或昂贵的 LLM 打分来过滤数据，而是用 programmatic editing（function-calling 框架）来精炼数据。这与 PPL-Factory 的"预算感知"选择、DRIFT 的"on-policy 归因"一起，标志着数据过滤正在从静态规则走向动态自适应系统。

### 2. 合成数据的质量保障成为热点
本期有 2 篇直接讨论合成数据质量（#3 model collapse, #10 environment-free），加上基线中已有的相关工作，合成数据的质量控制正在从"事后检测"转向"过程中防止"。

### 3. 跨模型/跨任务的数据评估
DataShield 的 consensus subspace 和 PPL-Factory 的 task-aware selection 都强调：数据质量不是一个绝对概念，而是取决于目标模型和目标任务。这与基线中 "A Bitter Lesson for Data Filtering"（7/20）的核心观点一致——没有放之四海皆准的过滤策略。

### 4. 此前遗漏论文较多
本期 10 篇中有 8 篇是此前遗漏的（非增量窗口内提交），说明：(1) DuckDuckGo 不可用时搜索覆盖率下降；(2) 5-7 月有一批高质量数据选择/课程学习论文发表，值得关注。

### 5. 与 7/20 报告的对比
7/20 报告的核心洞察是"过滤可能不如不过滤"（Bitter Lesson），本期则更关注"如何更好地过滤"——PPL-Factory、UltraX、DRIFT 都在寻找更智能的过滤/选择方法。两个视角互补：先认识到过滤的局限性，再在此基础上设计更精准的方法。
